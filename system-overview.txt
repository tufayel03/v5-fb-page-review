             FB PAGE REVIEW SYSTEM - ARCHITECTURE & CODE OVERVIEW

This file provides a comprehensive reference describing the Chrome Extension,
API routes, database schema, and scraping mechanisms used in this application.
It acts as the ultimate guide to help developers in future sessions hit the ground
running without requiring any manual explanation.

-----------------------------------------------------------------------------
1. DIRECTORY STRUCTURE & FILE ROLES
-----------------------------------------------------------------------------
The project is a Web Application (Vite + React frontend & Node.js + Express backend) 
integrated with a specialized SQLite database and supported by a custom Chrome Extension 
for administrative scraping / fraud monitoring.

A. Backend Core:
* /server.ts: The core entry point. Handles Express setup, API routing, moderator middlewares,
  image processing (Sharp), shortlink resolution, Facebook scraping, Google Sheets sync schedules,
  and DB transactions.
* /database.ts: Creates SQLite tables, enables WAL mode, configures indices on all primary/foreign keys,
  runs database migrations (e.g. adding columns), and auto-heals corrupted records on startup.
* /importService.ts: Handles bulk Excel/CSV row importing for pages and contact numbers.
* /business_endpoints.ts: Business owner features like updating business profiles, category management.
* /fraud_endpoints.ts: Extensible module for fraud logs.

B. Chrome Extension (/chrome-extension/):
* manifest.json: Configures standard activeTab, scripting, cookies, host permissions (facebook.com),
  and details popup files.
* content.js: Extracts page URL, name, and profile pictures from Facebook DOM (SVG image tags guest bypass).
* popup.html: Compact dark-theme admin UI (#050b18 background) matching fbpagereview.com design.
* popup.js: Handles login authentication, fetches current page state from backend database, pre-fills
  inputs, and submits status/contact details directly to the VPS.

C. React SPA Frontend (/src/):
* main.tsx & App.tsx: Entry points & React Router configuration.
* /src/pages/
  - Home.tsx: Landing page displaying the search bar, category grids, stats, and infinite marquee.
  - Dashboard.tsx: Public profile statistics and verified badges lists.
  - GlobalSearch.tsx: Consolidated directory search page.
  - PageProfile.tsx: Public detailed view of a Facebook Page (reviews, trust details, dispute stats).
  - WriteReview.tsx: User form for filing review details with images, amounts, and be-half options.
  - FraudDirectory.tsx & FraudList.tsx: Public indices showing blacklisted fraud entities.
  - Login.tsx, Register.tsx, ForgotPassword.tsx, ResetPassword.tsx: Auth components.
  - Blog.tsx & BlogPost.tsx: Public-facing blog layout.
* /src/pages/admin/ (Administrator Dashboard pages):
  - AdminDashboard.tsx: Overall stats panel (users, claims, reviews, pages).
  - AdminPages.tsx: Primary page directory list with URL-driven persistent pagination state.
  - AdminPageDetails.tsx: Edit panel for a page (change status, edit profile photo, manage contacts).
  - AdminContactNumbers.tsx: Contact number manager showing linked pages as links with accurate count badges.
  - AdminContactNumberDetails.tsx: Details view for a phone number with owner and page link mappings.
  - AdminReviews.tsx & AdminReviewDetails.tsx: Moderation table for incoming reviews.
  - AdminUsers.tsx: User account role configuration.
  - AdminPageClaims.tsx & AdminPageClaimDetails.tsx: Claim request review center.
  - AdminDisputes.tsx & AdminDisputeDetails.tsx: Disputes raised by page owners against reviews.
  - AdminMediaLibrary.tsx: Uploaded media resource files.
  - AdminLogs.tsx: Operational audit logs.
  - AdminSettings.tsx: System configs, security options, and database backup generator.
  - AdminGoogleSheetSync.tsx: Sheets sync configuration, auto-intervals, and raw logs.
  - AdminBulkImport.tsx: Upload module for Excel templates.
* /src/pages/business/ (Business Owner space):
  - BusinessLayout.tsx, BusinessOverview.tsx: Workspace wrapper & stats.
  - BusinessPages.tsx: Claims list of claimed pages.
  - BusinessReviews.tsx: Customer review answers.
  - BusinessContactNumbers.tsx: Business contact number manager.
  - BusinessSettings.tsx & BusinessProfileInfo.tsx: Business metadata editors.

2. COMPLETE DATABASE SCHEMA (SQLite WAL Mode)
-----------------------------------------------------------------------------
Below is the full table mapping configured in database.ts:

1. Users: Stores admin, moderator, business owner, and user accounts.
   - id (TEXT, PK): Unique UUID.
   - full_name (TEXT), username (TEXT, Unique), email (TEXT, Unique), password_hash (TEXT).
   - role (TEXT): 'admin', 'moderator', 'business_owner', 'user'.
   - reset_token (TEXT), reset_token_expires (DATETIME).
2. FacebookPages: The core table cataloging indexed Facebook profiles.
   - id (TEXT, PK): UUID.
   - current_name (TEXT): Canonical page name.
   - facebook_url (TEXT, Unique): Canonical URL.
   - profile_picture (TEXT): WebP filename (uploads/...).
   - status_badge (TEXT): 'Under Review', 'Verified Marketplace Seller', 'Suspicious', 'Reported as Fraud', 'Old/Dead Page'.
   - trust_score (INTEGER): -100 to 100 indicator.
   - is_fraud_listed (INTEGER): 0 or 1 indicator.
   - contact_number (TEXT), extra_contacts (TEXT), payment_methods (TEXT), page_details (TEXT).
   - website_url (TEXT), owner_id (TEXT, FK -> Users.id), claim_status (TEXT).
   - average_rating (REAL), total_reviews (INTEGER).
3. Reviews: User submissions about business transaction experiences.
   - id (TEXT, PK), page_id (FK), user_id (FK), star_rating (INTEGER).
   - title (TEXT), description (TEXT), status (TEXT: 'Approved', 'Pending', 'Rejected').
   - bkash_number (TEXT), order_amount (TEXT), proof_image (TEXT).
   - is_on_behalf (INTEGER), on_behalf_name (TEXT).
4. ContactNumbers: Aggregated phone numbers associated with one or more pages.
   - id (TEXT, PK), number (TEXT, Unique).
   - type (TEXT: 'bKash', 'Nagad', 'Rocket', 'Personal', 'Agent').
   - display_name (TEXT): User-defined name or page fallback name.
   - linked_page_ids (TEXT): Comma-separated list of UUIDs.
   - linked_page_count (INTEGER): Dynamic count of active pages.
   - status (TEXT: 'Normal', 'Suspicious', 'Reported').
   - admin_note (TEXT).
5. Claims: Page ownership claims filed by business users.
   - id (TEXT, PK), page_id (FK), user_id (FK), status (TEXT).
6. Disputes: Dispute requests filed against reviews.
   - id (TEXT, PK), page_id (TEXT), review_id (TEXT), status (TEXT).
7. OwnerReplies: Business owner comments on page reviews.
8. SearchQueries: Tracks user searches (query, count, updated_at).
9. GoogleSheetSyncSettings: Configurations for automated imports.
10. AdminLogs: Action details audit tracking.

3. CRAWLER PIPELINE & GUEST BYPASS ROADBLOCKS
-----------------------------------------------------------------------------
The central function is `scrapeAndAddFacebookPage(url)` in server.ts:

1. Canonical URL Conversion:
   - Legacy URLs (like `facebook.com/123` or `profile.php?id=123`) are converted into SEO-friendly format `/people/Page/id/`.
   - URL shortlinks (`facebook.com/share/X`) are followed via high-performance curl requests to parse final target URLs.
2. Deduplication:
   - Uses ID extraction to check database existence before executing scrapers, preventing duplication.
3. Multi-Tier Scraping Fallbacks:
   - Tier 1: Pulls the public Page Plugin widget iframe via curl to read clean HTML.
   - Tier 2: Crawls public SEO page directly using guest headers.
   - Tier 3: Passes crawls via a Google Translate proxy to bypass VPS IP restrictions.
   - Tier 4: Public Graph API redirects.
4. Image Optimization:
   - Validates images through `sharp().metadata()`. Invalid formats are skipped.
   - Saves processed avatars as optimized 300x300 WebP files.
5. Multilingual Login-Wall Detection:
   - Inspects page titles for keywords ("Log in", "लॉग इन", "সাইন আপ", "পাহण्यासाठी") and defaults page titles to the username to avoid saving junk names.
6. Exact Title Parsing:
   - Saves titles with exact dashes and punctuation intact (e.g. `Lovely Mart - লাভলি মার্ট`) without stripping symbols or forcing capitalization.

4. FRONTEND UX & PERSISTENT ROUTING
-----------------------------------------------------------------------------
* Persistent URL Query Params:
  - In `AdminPages.tsx`, pagination, filters, and searches are managed through React Router's `useSearchParams`.
  - Details views handle browser back navigation via `navigate(-1)` to preserve pagination context.
* Infinite Home Carousel:
  - Scrolling marquee list containing fraud cards animated via pure CSS `translate3d` (hardware-accelerated looping). Pauses on hover.
* Contact Number Mappings:
  - The contact table displays linked page names as active links.
  - Counts the number of active, non-deleted pages correctly in real-time.

-----------------------------------------------------------------------------
5. DELETION & DYNAMIC CLEANUP LOGIC (RECENT UPDATES v0.08 & v0.09)
-----------------------------------------------------------------------------
To ensure data is always synchronized and deleted pages do not leave ghost records:
1. Deletion Hook:
   - When a page is deleted from the Admin panel, `unlinkPagesFromContactNumbers(pageIds)` runs inside the SQLite transaction.
   - It updates all associated `ContactNumbers` rows, removing the deleted page ID, reducing the `linked_page_count`, and recalculating the phone number's status badge.
2. Dynamic Badge & Name:
   - Display counts and link list names rely on the actual existing pages returned from the database rather than legacy string lengths.
3. Database Startup Healer:
   - Automatically unescapes Unicode sequences and checks database rows on launch to purge invalid links and restore Bangla texts.
